# 粮仓深钻除板结机器人智能管控平台方案

## 1. 项目定位

本系统定位为一套纯软件演示型管控平台，用于展示粮仓内深钻除板结机器人的作业管理、设备监控、粮情预警和调度决策能力。当前阶段不接入真实硬件和真实业务接口，所有数据先使用 Mock 数据驱动，保留基础交互，让用户能够完成“查看态势、进入模块、切换仓号、查看任务、模拟控制、查看告警”的演示流程。

UI 风格参考当前目录下的四张界面图：

- `主界面.png`
- `仓库粮食信息.png`
- `仓库设备信息.png`
- `作业监控.png`

整体保持“深色科技大屏 + 荧光绿色线框 + 粮仓数字孪生 + 数据看板”的视觉方向，可在信息密度、按钮状态、图表可读性和响应式适配上做轻微优化。

## 2. 设计目标

1. 保持现有视觉识别：深绿黑底、发光边框、数字孪生粮仓、状态卡片、仪表盘、趋势图和视频缩略图。
2. 建立清晰的信息架构：主界面作为全局驾驶舱，子页面分别承载粮情、设备、作业、告警和任务管理。
3. 先做可演示版本：数据全部 Mock，交互以页面切换、仓号切换、任务筛选、告警确认、模拟控制为主。
4. 便于后续扩展：页面结构、Mock 数据结构和组件命名提前贴近真实业务，方便以后替换为接口数据。

## 3. 推荐系统名称

**国粮智卫 | 粮仓深钻除板结机器人智能管控平台**

该名称和当前主界面顶部标题一致，适合作为演示系统主标题。子系统可以拆分为：

- 粮仓作业管理
- 作业监控与控制
- 粮情监测与预警
- 仓库设备信息
- 报警管理与联动控制

## 4. 信息架构

### 4.1 顶层导航

建议采用顶部标题栏 + 主功能入口的结构。顶部固定展示品牌、系统状态、用户、时间；中部或底部提供模块入口。

页面建议如下：

| 页面 | 路由建议 | 说明 |
| --- | --- | --- |
| 主界面 | `/dashboard` | 全局态势总览，展示仓群、任务、风险、视频和核心指标 |
| 粮仓作业管理 | `/operations` | 任务管理、作业调度、机器人分配、仓储档案 |
| 作业监控与控制 | `/control` | 单次作业的数字孪生监控、参数控制、联动策略、视频回放 |
| 粮情监测与预警 | `/grain-monitor` | 温湿度、氧气、二氧化碳、粉尘、板结指数和风险评估 |
| 仓库设备信息 | `/devices` | 机器人、传感器、通信设备、能源系统、摄像头等设备台账 |
| 告警中心 | `/alarms` | 告警列表、告警等级、处理状态、联动记录 |

第一版可以先完成前 4 个页面，`devices` 和 `alarms` 可以作为弹窗、侧栏或简化页面存在。

## 5. 页面方案

### 5.1 主界面

主界面是系统第一屏，承担全局驾驶舱作用。

核心区域：

- 顶部标题栏：品牌 Logo、系统名称、系统状态、用户、当前时间。
- 中央粮仓数字孪生：展示 A01、A02、A03、B01、B02、B03 六个粮仓状态。
- 左侧状态面板：设备健康状态、环境监测、实时告警。
- 右侧任务面板：作业任务概览、当前作业、趋势与决策支持、视频预览。
- 底部入口与指标：粮仓作业管理、作业监控与控制、粮情监测与预警、机器人总数、今日作业时长、日产量、累计处理量。

基础交互：

- 点击仓号标签，切换当前选中仓，联动左右侧数据。
- 点击底部模块卡片，进入对应页面。
- 告警列表支持点击查看详情弹窗。
- 视频预览支持点击打开大图或模拟播放弹窗。
- 中央视图支持“3D 视图 / 热力图 / 风险分布”三种模式切换。

### 5.2 粮仓作业管理

该页面用于展示作业任务、仓储信息、机器人调度和设备运行概况。

核心模块：

- 顶部 KPI：在管粮仓、当前任务、在线机器人、今日调度、设备完好率。
- 仓储信息管理：仓号选择、仓型、粮种、总容量、当前仓储量、平均温湿度、板结等级。
- 粮仓数字孪生与深钻作业监控：突出当前作业仓，如 B02，展示作业点位。
- 任务管理：任务编号、仓号、任务类型、优先级、状态、计划时间。
- 作业调度：机器人分配、时间轴、任务排程。
- 设备管理：通信设备、能源系统、感知设备、机器人状态。
- 快捷入口：仓储档案、任务下发、调度策略、设备台账。

基础交互：

- 点击仓号列表，刷新仓储信息和容量图。
- 任务表支持按状态筛选：执行中、待执行、计划中、已完成。
- 点击“任务下发”弹出模拟表单，可选择仓号、任务类型、机器人和时间。
- 点击“调度策略”弹出策略建议面板。

### 5.3 作业监控与控制

该页面用于展示机器人在单仓内的实时作业过程，是系统演示重点。

核心模块：

- 中央作业数字孪生：粮仓剖面、机器人位置、作业轨迹、目标深度和作业区域。
- 左侧实时监控：位置坐标、当前深度、作业进度、覆盖率、姿态角、行进速度。
- 作业环境感知：粮面扰动热力图、不稳定区域识别、板结识别分布图。
- 右侧负载监控：钻进阻力、扭矩、电流趋势。
- 参数控制：作业深度、破碎力度、行进速度。
- 远程控制：远程接管、开始作业、暂停作业、急停、撤离、复位。
- 异常检测：卡滞、陷落风险、倾覆趋势、姿态异常。
- 底部视频可视化：实时视频、侧视视频、作业过程回放。

基础交互：

- “开始作业 / 暂停作业”切换作业状态。
- 参数控制按钮支持加减数值，刷新 Mock 状态。
- 急停按钮触发红色风险状态和告警记录。
- 点击视频区域打开模拟播放器。
- 联动策略按钮可切换自动减速、绕行、路径重规划、区域更新、策略调整等状态。

### 5.4 粮情监测与预警

该页面用于展示粮仓环境、粮情变化和综合风险等级。

核心模块：

- 顶部环境 KPI：平均粮温、平均湿度、氧气、二氧化碳、粉尘浓度、综合风险。
- 中央空间风险分布：粮仓 3D 热力图，展示低风险、轻度风险、中度风险、高风险区域。
- 左侧粮情与环境监测：温度趋势、湿度趋势、气体监测、粉尘浓度、板结指数趋势。
- 右侧风险识别：燃爆风险、霉变风险、气体危害风险、粉尘堆积风险、结构风险、设备风险。
- 动态阈值：各指标阈值、当前值、趋势方向。
- 决策建议：关注高风险仓、启动通风、调整作业频次、执行巡检。
- 底部报警管理：等级统计、联动策略、报警记录与追溯、数据视频。

基础交互：

- 点击风险等级图例，筛选中央热力显示。
- 告警卡片支持确认处理，状态从“处理中”变为“已处理”。
- 趋势图支持近 7 天、近 24 小时切换。
- 决策建议支持点击“执行”，写入模拟处理记录。

### 5.5 仓库设备信息

该页面可在第一版做成简化页或设备弹窗。

核心模块：

- 机器人列表：编号、所在仓、在线状态、电量、任务状态、故障状态。
- 传感器列表：温度、湿度、氧气、二氧化碳、粉尘、振动、摄像头。
- 通信设备：网关、基站、信号强度、在线率。
- 能源系统：电池组、充电桩、供电状态。
- 维护记录：最近巡检、故障次数、保养提醒。

基础交互：

- 按设备类型筛选。
- 点击设备查看详情。
- 模拟启停、复位、报修。

## 6. Mock 数据设计

### 6.1 数据原则

- 所有数据固定在前端本地，便于离线演示。
- 数据结构按真实接口思路设计，后续替换 API 时成本低。
- 数值变化可以用定时器轻微波动，增强“实时感”。
- 告警、任务、设备状态允许用户操作后修改本地状态。

### 6.3 模拟实时变化

建议每 2 到 5 秒更新部分指标：

- 温度在 `±0.2` 范围内浮动。
- 湿度在 `±0.5` 范围内浮动。
- 作业进度缓慢增加。
- 负载、电流、扭矩曲线随机生成最近 60 秒点位。
- 当用户点击“急停”或“执行建议”时，立即写入告警和日志。

## 7. UI 视觉规范

### 7.1 色彩

保持当前参考图的主视觉：

| 用途 | 建议颜色 |
| --- | --- |
| 页面背景 | `#020A06`, `#03170D` |
| 主荧光绿 | `#55FF7A`, `#7CFF4F` |
| 边框绿 | `rgba(76, 255, 115, 0.55)` |
| 面板底色 | `rgba(4, 35, 18, 0.88)` |
| 成功状态 | `#7CFF4F` |
| 警告状态 | `#FFD43B` |
| 危险状态 | `#FF4D3D` |
| 信息蓝绿 | `#35D7FF` |
| 普通文本 | `#E8FFE8` |
| 次级文本 | `#A7D7A8` |

### 7.2 布局

- 目标分辨率优先适配 `1920x1080` 大屏。
- 面板使用细线发光边框，圆角控制在 `4px-8px`。
- 主界面中央数字孪生占最大视觉权重。
- 左右两侧面板保持稳定网格，避免信息跳动。
- 图表区域不要过度装饰，保证数字和趋势清晰。

### 7.3 字体与数字

- 中文标题建议使用思源黑体、阿里巴巴普惠体或系统黑体。
- 大数字使用更粗字重，突出运行态势。
- 单位小字号展示，紧贴指标数字。
- 告警等级、状态、优先级使用颜色块或描边标签。

### 7.4 图表

建议使用 ECharts：

- 折线图：温度、湿度、板结指数、负载、电流、扭矩。
- 仪表盘：任务完成率、仓容利用率、风险分值。
- 环形图：设备状态、机器人状态。
- 柱状图：各仓容量、任务数量、告警统计。
- 热力图：风险分布和板结分布。

### 7.5 数字孪生表现

第一版建议使用静态 3D 图片或轻量 Canvas 效果，不强制做真实 3D。可以通过以下方式增强演示感：

- 仓体外框呼吸光效。
- 当前作业仓高亮。
- 机器人点位闪烁。
- 作业轨迹用环形或螺旋线动画。
- 风险区域使用半透明色块叠加。

后续如需升级，可使用 Three.js 实现可旋转的粮仓模型。

## 8. 交互流程

## 8.1 全局仓库联动机制

除主界面外，粮仓作业管理、作业监控与控制、粮情监测与预警三个核心子界面也需要支持按仓库切换，并联动刷新当前页面内的业务数据。建议系统维护一个全局选中仓库状态 `selectedWarehouseId`，用户在任意页面切换仓库后，其它页面也同步记住该仓库。

例如：用户在主界面选择 `B02`，进入“作业监控与控制”页面时默认展示 `B02` 的机器人、作业进度、负载曲线、视频和告警；如果再切换到 `A02`，则页面立即刷新为 `A02` 的任务状态和监控数据。若目标仓当前没有作业，则展示“待机 / 暂无任务”状态，同时保留环境数据、设备状态和历史告警。

各子页面联动内容如下：

| 页面 | 仓库切换后联动内容 |
| --- | --- |
| 粮仓作业管理 | 仓储信息、仓容利用率、粮食品类、当前任务、机器人分配、作业计划、设备状态、近期日志 |
| 作业监控与控制 | 当前机器人、作业状态、作业深度、目标深度、进度、覆盖率、负载曲线、异常检测、视频画面、控制参数 |
| 粮情监测与预警 | 温湿度、氧气、二氧化碳、粉尘浓度、板结指数、风险热力图、风险等级、告警记录、决策建议 |

第一版可以通过前端全局状态实现：

```js
const appState = {
  selectedWarehouseId: "B02",
  currentPage: "dashboard"
};
```

所有 Mock 数据均按 `warehouseId` 建立关联。页面渲染时根据 `selectedWarehouseId` 过滤或聚合数据，保证主界面和三个子界面形成一致的演示闭环。

### 8.2 演示主流程

1. 进入主界面，查看全局仓群、设备和任务态势。
2. 点击 B02 仓，查看该仓风险、作业任务和视频预览。
3. 点击“作业监控与控制”，进入深钻作业监控页面。
4. 点击“开始作业”，作业进度和曲线开始模拟变化。
5. 调整作业深度、破碎力度、行进速度。
6. 点击“急停”，系统生成高风险告警。
7. 进入“粮情监测与预警”，查看 B02 风险详情和决策建议。
8. 点击“执行建议”，告警状态更新为已处理。

| 功能 | 第一版实现方式 |
| --- | --- |
| 页面切换 | 前端路由或 Tab 切换 |
| 仓号选择 | 本地状态切换 |

## 10. 开发阶段规划

### 第一阶段：静态高保真页面

目标：完成主界面、粮仓作业管理、作业监控与控制、粮情监测与预警的静态布局。

产出：

- 页面框架
- 主题样式
- 面板组件
- Mock 数据结构
- 图表占位与基础图表

### 第二阶段：基础交互

目标：让演示流程能够跑通。

产出：

- 页面路由
- 仓号切换
- 任务筛选
- 作业启停
- 参数加减
- 告警确认
- 视频弹窗

### 第三阶段：动态演示增强

目标：提升实时感和展示效果。

产出：

- 定时刷新 Mock 数据
- 曲线动态滚动
- 作业轨迹动画
- 仓体高亮和风险热力叠加
- 日志自动追加

### 第四阶段：接口替换准备

目标：为真实后端或硬件接口预留边界。

## 11. 需要准备的素材

第一版可以直接使用当前参考图中的视觉方向重新绘制界面。如果希望更贴近真实系统，建议补充：

- 国粮智卫 Logo 源文件
- 粮仓 3D 模型或透明 PNG
- 深钻机器人透明 PNG 或模型
- 粮仓内部视频截图
- 设备图标
- 告警图标
- 仓库现场照片

没有素材时，可以先用现有截图风格的静态图片、CSS 发光线框和 ECharts 图表完成演示。

## 12. 验收标准

第一版演示系统建议按以下标准验收：

1. 页面风格与参考图一致，整体保持深色科技大屏效果。
2. 主界面能展示仓群态势、设备状态、任务进度、风险和视频入口。
3. 至少 4 个核心页面可切换。
4. 仓号选择、任务筛选、作业启停、参数调整、告警处理可交互。
5. 所有数据来自本地 Mock，无需后端和硬件。
6. 页面在 `1920x1080` 分辨率下无明显遮挡、溢出和错位。
7. 关键数字、风险等级、告警状态清晰可读。

## 13. 第一版优先级

建议优先级如下：

| 优先级 | 内容 |
| --- | --- |
| P0 | 主界面、作业监控与控制、Mock 数据、基础页面切换 |
| P1 | 粮仓作业管理、粮情监测与预警、任务筛选、告警处理 |
| P2 | 设备信息、视频弹窗、动态曲线、轨迹动画 |
| P3 | Three.js 数字孪生、复杂调度策略、真实接口预留 |

## 14. 总结

本系统第一版应以“可演示、风格统一、交互清楚”为核心，不需要一开始追求真实 3D 或真实设备接入。视觉上保持参考图的深绿科技大屏风格，功能上围绕粮仓、机器人、任务、风险、告警五类对象建立 Mock 数据和交互闭环。这样既能快速完成演示版本，也能为后续接入硬件、算法和业务系统留下清晰扩展空间。
